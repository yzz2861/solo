第二轮重点修正老面计算逻辑，保留现有界面和配方缩放功能。

当前环境面板里的“老面比例”没有真正参与老面加入量和扣水量计算，只影响了酵母减量和边界提示。需要把它接入核心配方计算：用户调整老面比例后，最终老面重量、水量扣减、面粉/水的平衡结果都应随之变化，而不是继续只按基础配方里的老面行缩放。

同时修正老面含水量折算方式。现在 `waterInStarter = scaledStarter * hydration / 100` 会把老面总重直接按水合率扣水，语义不对。比如 65% 水合的老面，应按“面粉 + 水 = 老面总重”的配方关系折算其中水分，再用于主配方扣水，避免学徒按错误水量称量。

完成后要保证项目可安装、可运行，并能通过实际调整老面比例和水合率验证结果变化是否正确。
