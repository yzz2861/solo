## 已实现的业务规则

1. **衣物已取走后不可改收衣状态** — `picked_up` 状态下禁止添加备注和修改状态，返回 `ORDER_PICKED_UP` 错误
2. **同一订单重复申请合并处理** — 对已有 pending 申请的订单再次提交，自动将金额合并到首条申请，状态标记为 `merged`
3. **赔付金额超过阈值需高级复核人** — 金额 > 500 元时 `requiresSeniorReview=true`，非高级复核人 approve 会被拒绝（`SENIOR_REVIEW_REQUIRED`）
4. **客服时间线** — 每个操作（创建订单、添加备注、提交赔付、合并、复核、补充材料）都写入 timeline，按时间排序
5. **财务导出与订单状态一致** — 导出数据同时包含 `claimStatus` 和 `orderStatus`，CSV 格式含 BOM 头方便 Excel 打开
6. **门店一次性批量补充照片** — `supplements` 接口支持 photos 数组，一次提交多张照片，避免顾客久等
