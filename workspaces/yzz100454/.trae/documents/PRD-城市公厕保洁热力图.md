## 1. 产品概述

城市公厕保洁热力图系统，整合巡检记录、保洁打卡、客流统计和群众投诉四类数据，按点位和时段合并分析，帮助城管队科学调配保洁力量。

- **核心问题**：公厕管理数据分散、节假日加人靠经验、缺乏量化考核依据
- **目标用户**：城管队长、环卫所长、保洁班长、街道考核人员
- **产品价值**：数据驱动的保洁排班、可追溯的考核依据、异常点位主动发现

## 2. 核心功能

### 2.1 用户角色

| 角色 | 使用方式 | 核心权限 |
|------|----------|----------|
| 城管队长 | 热力图总览、异常点位查看 | 查看全量数据、调班建议、导出数据 |
| 环卫所长 | 周报导出、考核数据 | 导出周报、查看考核指标 |
| 保洁班长 | 天气/活动回看、临时加巡建议 | 历史数据回看、加巡建议 |
| 街道考核人员 | 考核数据查看 | 查看考核依据数据 |

### 2.2 功能模块

1. **热力图总览页**：点位地图分布、时段热力条、异常标记
2. **数据管理页**：四类数据（巡检/保洁/客流/投诉）的导入与浏览
3. **异常分析页**：高客流低保洁、多投诉巡检正常、连续缺打卡三类异常
4. **报告导出页**：周报生成与导出、考核数据导出
5. **历史回看页**：按天气/活动日筛选、时段对比、临时加巡建议

## 3. 核心流程

### 3.1 日常巡检流程
保洁人员打卡 → 系统记录时间与点位 → 与客流数据合并 → 生成热力图 → 队长查看异常 → 调整排班

### 3.2 异常识别流程
导入多源数据 → 点位名称归一化 → 按时段聚合统计 → 阈值比对 → 标记异常点位 → 推送告警

### 3.3 周报生成流程
选择时间范围 → 自动汇总四类数据 → 生成分析图表 → 计算考核指标 → 导出PDF/Excel

    E --> F["热力图展示"]
    E --> G["异常告警列表"]

## 4. 用户界面设计

### 4.1 设计风格
- **主色调**：深蓝(#1e3a5f) + 青蓝(#0ea5e9)，体现城市管理专业感
- **辅助色**：热力渐变色（绿→黄→橙→红），直观展示保洁压力
- **按钮风格**：圆角中等，悬停有微动效，主次按钮区分明确
- **字体**：中文使用思源黑体，英文使用Inter，清晰易读
- **布局风格**：左侧导航 + 右侧内容区，卡片式信息分组
- **图标风格**：线性图标，统一2px描边，与整体风格协调

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|-----------|-------------|---------|
| 热力图总览 | 地图区 | 全屏地图、热力点标记、弹窗详情 |
| 热力图总览 | 顶部统计 | 四个数据卡片、渐变背景、数字动画 |
| 热力图总览 | 时间控制 | 时间滑块、播放/暂停、时段切换 |
| 数据管理 | 左侧分类 | Tab切换四类数据 |
| 数据管理 | 右侧表格 | 数据表格、导入按钮、筛选条件 |
| 异常分析 | 三列卡片 | 三类异常各一列，可展开查看详情 |
| 报告导出 | 预览区 | 报告预览、样式模板选择 |
| 历史回看 | 双栏对比 | 左右两个时间轴对比视图 |

### 4.3 响应式
- 桌面端为主设计，适配1440px及以上
- 平板端折叠侧边栏为图标模式
- 移动端简化为列表视图，地图缩小为缩略图

### 4.4 动效设计
- 页面加载：卡片错峰淡入，0.1s间隔
- 热力点：呼吸动画，异常点位频率更快
- 切换页面：内容区淡入淡出过渡
- 悬停反馈：按钮上浮+阴影加深
