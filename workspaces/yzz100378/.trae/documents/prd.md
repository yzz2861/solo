## 1. 产品概述

地铁换乘拥堵调度是一款面向地铁站务人员的培训模拟游戏。玩家在站厅地图上布置围栏、开关扶梯、放置引导员，让不同方向客流在限定时间内安全高效通过换乘大厅。系统通过实时客流模拟、突发事件响应和赛后回放分析，帮助新人快速掌握拥堵调度技能，站长可按真实站点配置关卡并追踪训练成绩。

- **目标用户**：地铁新入职站务员（玩家）、站长/培训主管（管理员）
- **核心价值**：降低早高峰换乘拥堵处理的学习成本，用交互式模拟替代纯理论培训，让新人在安全环境中积累调度经验

## 2. 核心功能

### 2.1 用户角色

| 角色 | 入口方式 | 核心权限 |
|------|----------|----------|
| 站务员（玩家） | 直接进入游戏 | 选择关卡、进行调度游戏、查看回放与成绩 |
| 站长（管理员） | 管理员入口 | 配置关卡、查看训练成绩统计、分析薄弱环节 |

### 2.2 功能模块

1. **游戏主页**：关卡选择、成绩总览、快速开始
2. **调度游戏页**：站厅地图、客流模拟、工具面板、实时状态
3. **关卡编辑器页**：地图编辑、出入口配置、事件编排
4. **回放分析页**：赛后回放、瓶颈标注、绕行统计
5. **成绩管理页**：训练记录、薄弱点分析、趋势图表

## 3. 核心流程

### 3.1 站务员游戏流程

站务员进入游戏主页 → 选择关卡 → 查看关卡简介和地图预览 → 开始游戏 → 系统开始客流模拟，玩家布置围栏/开关扶梯/放置引导员 → 突发事件触发时调整策略 → 限定时间结束 → 系统评分并生成回放 → 查看回放分析（瓶颈、绕行） → 返回主页或重试

### 3.2 站长配置流程

站长进入管理入口 → 打开关卡编辑器 → 绘制站厅地图（墙壁、通道、出入口、扶梯、换乘口） → 配置各出入口客流方向和密度 → 编排突发事件（扶梯临停、出口关闭、客流突增） → 保存关卡 → 可选：导入/导出 JSON 配置 → 查看站务员训练成绩 → 分析薄弱环节

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：深蓝灰（#1a1f36）为底色，搭配地铁线路标志色（红 #e63946、蓝 #457b9d、绿 #2a9d8f）作为功能色
- **辅助色**：拥堵红（#ff4444）、绕行黄（#ffaa00）、通畅绿（#00cc66）用于客流状态
- **按钮风格**：圆角矩形，带轻微投影，hover 时放大 1.02 倍
- **字体**：标题用 Noto Sans SC Bold，正文用 Noto Sans SC Regular，数据用 JetBrains Mono
- **布局风格**：左侧工具面板 + 中央地图 + 右侧状态面板的三栏布局
- **图标风格**：线性图标，配合 lucide-react 图标库

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|---------|
| 游戏主页 | 关卡列表 | 卡片网格布局，每张卡片含站点缩略图、难度星级、完成勾标，hover 放大 |
| 游戏主页 | 成绩概览 | 折线迷你图 + 总分大字，背景渐变 |
| 调度游戏页 | 站厅地图 | Canvas 渲染网格地图，支持缩放平移，乘客以彩色圆点呈现 |
| 调度游戏页 | 工具面板 | 左侧竖排图标按钮组，选中态高亮，拖拽放置围栏/引导员 |
| 调度游戏页 | 实时状态 | 右侧面板，数字动态跳动，进度条动画 |
| 调度游戏页 | 事件通知 | 顶部滑入横幅，红底白字，3秒后自动消失 |
| 关卡编辑器页 | 地图绘制 | 中央 Canvas 画布，左侧工具栏（墙壁/通道/出入口/扶梯），点击网格放置 |
| 回放分析页 | 回放播放器 | 底部时间轴滑块，播放/暂停/倍速按钮，热力图叠加层 |
| 成绩管理页 | 薄弱点分析 | 雷达图 + 列表，红色标注低于均值的维度 |

### 4.3 响应式设计

- 桌面优先（1280px+），三栏布局
- 平板（768px-1279px），工具面板折叠为底部抽屉
- 手机（<768px），单栏布局，地图全屏，工具面板为悬浮按钮

### 4.4 动效设计

- 页面切换：淡入淡出 300ms
- 乘客流动：Canvas 逐帧平滑移动，拥堵时脉冲闪烁
- 围栏放置：弹性缩放动画
- 事件通知：从顶部滑入
- 评分结算：数字滚动递增效果
