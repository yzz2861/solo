## 1. 产品概述

温室灌溉蒸散估算系统是一款面向设施农业的专业决策支持工具，帮助番茄种植户和农技人员基于 FAO Penman-Monteith 公式科学计算每日需水量，避免裂果与萎蔫问题。

- 核心目标：通过多源传感器数据融合+作物系数法，提供可信赖的灌溉决策建议
- 目标用户：温室番茄种植户（农户版）、农业技术推广人员（农技员版）
- 市场价值：替代经验灌溉，降低裂果率15%~30%，节水20%以上

## 2. 核心功能

### 2.1 用户角色

| 角色 | 权限说明 | 报告呈现方式 |
|------|---------|-------------|
| 种植户 | 数据录入、查看建议、回填实际浇水量 | 农户版：简洁人话、操作指引 |
| 农技员 | 以上全部 + 查看计算过程、参数复核 | 农技员版：保留全部参数与计算步骤 |

### 2.2 功能模块

1. **数据输入页**：气象参数录入、作物信息填写、土壤条件输入、灌溉系统配置
2. **估算报告页**：双版本报告切换、需水量展示、灌溉时段建议、异常提示面板
3. **历史记录页**：每日数据回填、建议值 vs 实际值对比图表、周度趋势分析

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|---------|---------|---------|
| 数据输入页 | 气象参数组 | 气温（℃/℉切换）、相对湿度（%）、光照强度（W/m² / lux 切换）、风速（m/s / km/h 切换） |
| 数据输入页 | 作物与土壤组 | 作物阶段下拉（未选时标黄提示）、土壤湿度（%体积含水量 / mbar 张力切换） |
| 数据输入页 | 灌溉系统组 | 灌溉效率滑块（默认80%）、灌水方式选择（滴灌/微喷/沟灌） |
| 数据输入页 | 异常检测面板 | 实时校验：缺测项标红、湿度跳变>30%标橙、作物阶段未选提示保守估算 |
| 估算报告页 | 双版本切换 | Tab 切换农户/农技员视图，农户版隐藏中间计算量 |
| 估算报告页 | 需水量卡片 | 展示 ET₀（参考蒸散）、ETc（作物蒸散）、净灌溉量、毛灌溉量，单位 mm/亩·方 |
| 估算报告页 | 时段建议 | 基于温度+光照推算最佳灌溉窗口（清晨/傍晚/分时段），附理由说明 |
| 估算报告页 | 保守提示条 | 顶部固定，列出数据异常项及估算偏保守的幅度 |
| 历史记录页 | 回填表单 | 选择日期、录入实际灌水量、备注（如"今天下雨"） |
| 历史记录页 | 对比图表 | 柱状图：建议量 vs 实际量；折线图：周度累计偏差趋势 |
| 历史记录页 | 周度总结 | 统计本周偏差率、节水/超水量、下周调整建议 |

## 3. 核心流程

主流程：用户录入当日传感器数据 → 系统校验异常并标注 → 调用 Penman-Monteith 引擎计算 → 结合作物阶段 Kc 系数 → 生成双版本报告 → 用户可回填实际灌水量 → 下周查看偏差分析

```mermaid
flowchart LR
    A["录入传感器数据"] --> B["异常检测校验"]
    B --> C["缺测/跳变/阶段未选?"]
    C -->|是| D["标注保守提示 + 估算修正系数"]
    C -->|否| E["使用标准参数计算"]
    D --> F["Penman-Monteith 计算 ET₀"]
    E --> F
    F --> G["乘以作物系数 Kc → ETc"]
    G --> H["考虑土壤湿度 → 净灌溉量"]
    H --> I["除以灌溉效率 → 毛灌溉量"]
    I --> J["生成灌溉时段建议"]
    J --> K{"选择报告版本"}
    K -->|农户版| L["人话描述 + 操作指引"]
    K -->|农技员版| M["参数列表 + 分步公式 + 结果"]
    L --> N["回填实际浇水量"]
    M --> N
    N --> O["入库存储"]
    O --> P["下周生成偏差对比图表"]
```

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：温室大棚绿 `#2D5A27`（稳重、农业感），配辅助色土壤棕 `#8B5E34`
- **高亮色**：警示橙 `#E67E22`（数据异常）、节水蓝 `#2980B9`（灌溉建议）
- **中性色**：浅米白底 `#FAF8F5`（纸张质感），深灰文字 `#2C3E50`
- **按钮风格**：圆角矩形（10px）、实心绿主按钮带微阴影、悬停上浮 2px
- **字体方案**：标题用「思源宋体 Bold」体现专业感，正文用「思源黑体 Regular」保证可读性
- **布局风格**：卡片式分组，气象/作物/灌溉三大块用带图标的顶部导航切换
- **图标风格**：Lucide 线性图标，关键数字旁配 emoji（🌡️💧☀️💨🍅）增强识别度

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI 元素要点 |
|---------|---------|------------|
| 数据输入页 | 顶部卡片 | 当日日期大标题，右侧显示"距上次灌溉 X 天" |
| 数据输入页 | 气象参数 | 数字输入框 + 单位下拉组合，左侧绿色图标，异常时右边缘变色 |
| 数据输入页 | 作物阶段 | 5 段式时间线（育苗→开花→坐果→膨果→成熟），点击选中 |
| 数据输入页 | 保守提示条 | 顶部固定，橙底白字，点击展开每项说明 |
| 估算报告页 | 水量卡片 | 大号数字展示，单位可切换（mm ↔ 方/亩），渐变进度条 |
| 估算报告页 | 时段建议 | 24h 时间轴热力图，标注推荐窗口（早5-7 / 晚18-20），附原因 |
| 估算报告页 | 版本切换 | Tab 切换带滑动下划线动画，农技员版用代码块风格展示公式 |
| 历史记录页 | 回填区 | 日期选择器 + 数字输入，"保存"按钮带成功打勾动画 |
| 历史记录页 | 对比图 | 双轴柱状图（建议绿 / 实际蓝），鼠标悬浮显示差值百分比 |
| 历史记录页 | 周总结卡 | 顶部进度条显示"节水 X%"或"超水 Y%"，底部文字建议 |

### 4.3 响应式

- **Desktop 优先**：1200px 断点下使用 2 列布局（输入左/结果右）
- **平板 768px**：折叠为单列，顶部标签切换输入/结果
- **手机 480px**：全单列，卡片间距收紧至 12px，字号缩小一级
- **触摸优化**：所有按钮最小 44px 高，输入框聚焦时键盘避让

### 4.4 动效与交互

- 页面加载：卡片依次上浮淡入（延迟 100ms 递增）
- 数据输入：错误提示带轻微抖动（3 次/200ms）
- 计算按钮：点击后 1.2s 进度环动画，随后结果卡片展开
- 报告切换：版本切换时内容区 0.3s 交叉淡入淡出
- 数字变化：需水量结果更新时用数字滚动动画（1s 内过渡到目标值）
