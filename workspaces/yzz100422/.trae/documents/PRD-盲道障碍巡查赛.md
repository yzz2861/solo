## 1. 产品概述

"盲道障碍巡查赛"是一款面向公益社团志愿者的盲道障碍识别培训应用。通过游戏化的街区地图巡查训练，帮助新志愿者快速掌握识别盲道障碍的技能，区分真正的通行障碍与不影响通行的旁侧物体，并学会正确的上报流程。

- **目标用户**：公益社团新志愿者（玩家）、带队老师（管理员）
- **核心价值**：降低实地培训成本，标准化培训内容，追踪个人薄弱点
- **应用场景**：志愿者上岗前培训、社团日常训练、薄弱环节针对性强化

## 2. 核心功能

### 2.1 用户角色

| 角色 | 进入方式 | 核心权限 |
|------|----------|----------|
| 志愿者 | 直接进入 | 参与巡查训练、查看个人进度与薄弱类型 |
| 带队老师 | 顶部入口切换 | 配置街区与障碍、查看街区错漏统计、管理训练数据 |

### 2.2 功能模块

1. **首页/关卡选择**：街区列表、训练进度展示、薄弱类型提示
2. **巡查游戏页**：街区地图、盲道路线、障碍点交互、判定面板
3. **结算页**：漏掉障碍统计、误报统计、得分与评级、知识点回顾
4. **个人进度**：历史成绩、薄弱类型分析、训练日历
5. **街区配置（老师端）**：添加/编辑街区、布置障碍点、设置判定规则
6. **统计分析（老师端）**：按街区错漏统计、薄弱路段排序、导出数据

## 3. 核心流程

### 3.1 志愿者训练流程

志愿者进入应用 → 选择街区关卡 → 进入巡查游戏 → 在地图上点击疑似障碍点 → 弹出判定面板选择答案 → 系统即时反馈（正确/错误+解释）→ 完成全部障碍或超时 → 结算页面展示漏掉和误报的点 → 记录成绩和薄弱类型 → 返回首页或继续下一关

```mermaid
flowchart LR
    A["进入首页"] --> B["选择街区关卡"]
    B --> C["进入巡查游戏"]
    C --> D["点击地图疑似障碍点"]
    D --> E["弹出判定面板"]
    E --> F["选择绕行/紧急程度/联系部门"]
    F --> G["即时反馈对错+解释"]
    G --> H["全部障碍完成?"]
    H -->|否| D
    H -->|是| I["结算页面"]
    I --> J["记录成绩与薄弱类型"]
    J --> K["返回首页或下一关"]
```

### 3.2 老师配置流程

老师进入配置模式 → 选择或创建街区 → 上传或绘制地图底图 → 在地图上布置障碍点 → 设置每个障碍的类型、判定规则、特殊说明 → 保存街区配置 → 返回统计页面查看训练数据

## 4. 用户界面设计

### 4.1 设计风格

- **设计方向**：温暖公益风 + 游戏化激励
- **主色调**：暖黄色（#F5A623，盲道砖色）作为主色，深青色（#2C5F6B）作为专业辅助色
- **辅助色**：绿色（#4CAF50）表示可绕行/安全，红色（#E53935）表示紧急/危险，橙色（#FF9800）表示需注意
- **按钮风格**：圆角矩形，有轻微阴影，hover 时有上浮效果
- **字体**：中文使用 Noto Sans SC，标题加粗，正文常规
- **布局风格**：卡片式布局，内容区域有圆角和柔和阴影
- **图标风格**：线性图标，统一 2px 线条，配色与主题一致
- **特殊元素**：盲道纹理作为装饰性背景元素，强化主题认知

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI 元素 |
|---------|---------|---------|
| 首页 | 街区列表 | 卡片网格布局，每张卡片含街名、进度条、薄弱标签、最佳成绩角标 |
| 首页 | 进度概览 | 顶部横幅，展示训练数据统计，有微动效 |

| 巡查游戏页 | 顶部状态栏 | 计时、已标记数、提示按钮、退出按钮 |
| 结算页 | 成绩总览 | 大字号得分、圆形进度条、评级徽章 |

### 4.3 响应式

- **桌面优先**设计，主界面最佳显示宽度 1280px 以上
- **平板适配**：1024px 时侧边栏收起为图标模式，卡片布局调整为 2 列
- **手机适配**：768px 以下单列布局，判定面板改为全屏弹出，地图支持双指缩放
- 触摸设备优化：按钮最小触控区域 44px，重要操作有明显视觉反馈

### 4.4 动效设计

- 页面切换：淡入 + 轻微上移动画
- 障碍点标记：点击后有波纹扩散效果
- 正确反馈：绿色对勾缩放出现，伴随轻微震动
- 错误反馈：红色叉号抖动出现
- 结算页：数据数字滚动动画，评级徽章从中心放大出现
- 盲道纹理：缓慢流动动画，增强视觉吸引力
